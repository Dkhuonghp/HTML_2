その2 親セレクタの参照

参照しているセレクタから親を指定したい場合、アンパサンド（&）を使用すると参照することが出来ます。
一番上の親からネストをしていくと、スタイルのネストが増え、可読性が落ちてしまうのをふせぎます。

Sample Code - 親参照を使わなかった場合

    #top{
        width: 100%;
        ul,ol{
            list-style: none;
        }
    }
    body.subPage #slide{
        margin: 0 20px;
    }

こういった場合、#slide内のネストやルールが増えると、下に落ちて可読性が下がる。

Sample Code - 親参照を使った場合

    #top{
        width: 100%;
        body.subPage &{
            margin: 0 20px;
        }
        ul,ol{
            list-style: none;
        }
    }

&を使って親を参照することが可能。

その３ プロパティのネスト

Sassのネストはセレクタだけでなく、プロパティにもセットすることが可能です。
プロパティのネストとはショートハンド（省略記述）で書けるプロパティで使うことができます。

Sample Code - 通常記述するCSS

    .sample{
        border-bottom: 1px solid #333;
        border-left: 5px solid #333;
    }

プロパティをネストすると以下の記述となります。

Sample Code - ネストされたプロパティ

    .sample{
        border:{
            bottom: 1px solid #333;
            left:{
                width: 5px;
                style: solid;
                color: #333;
            }
        }
    }
    
    //他にも
    .box{
        margin: 0{
            left: 20px;
        }
        border: 1px solid #999{
            bottom: 0;
        }
    }

コロン（：）の後に中括弧記述でネストすることで、border-bottomに対して一括指定や leftをさらにネストし、固有のスタイルを指定することが可能です。
また、値を続けて記述することで上書きすることも可能です。
CSSのプロパティにハイフン（ - ）があるものはすべてネストすることが可能です。

その5 変数

Sassでは、javaScriptと同じく変数を扱うことが可能です。ドルマークを使用し、変数名を定義します。
使用する際もドルマーク付きで記述することで、値を参照することが可能です。

Sample Code - 変数

    $mainColor: #990033;
    h1{
        background-color: $mainColor;
    }
    .topicTitle{
        cololr: $mainColor;
    }

テーマカラーなどよく使用するカラーコードなど汎用的な値は変数にすることで、値の変更時など 一括で指定することが出来ます。
また、@importルールで読み込むファイルに変数を記述し、読み込むことも可能です。

【練習】変数の練習

以下の内容にsubColor変数を作成し、practice05クラス内にあるそれぞれのセレクタの文字色を変更してみましょう。

Sassの変数

Sassには変数を扱うことができ、汎用的な値を格納・参照することが可能です。
変数を作成する際は、 $変数名 : 値 ; で作成し、変数名を記述することが使用が可能です。

変数のスコープ（参照可能な範囲）

変数はソースコード、上から処理される為、必ず使用したい場所より上で記述する必要があります。
一度作成した変数は、以降どこからでも参照可能ですが、ネスト内などで変数を作成した場合、ネスト外からの参照は出来ないので注意が必要です。

Sample Code - 変数のスコープ

    .contentWrap{
        $value: 20px;
        margin: $value;
        padding: $value;
        p{
            padding: $value;
        }
    }
    .box{
        //これはネスト外なので、参照することが出来ない。
        margin: $value;
    }

上記サンプルコードの様に、ネスト内で変数を作成すれば、汎用的な変数名を作成することが可能なので、
バッティングを回避することが出来ます。
もし、どちらも参照出来る変数名があった場合は、一番近い（後から記述）変数が上書きされ、参照されます。

変数で使用出来ない記号

変数は英数字、_（アンダースコア）-（ハイフン）を使用することが可能です。
また、マルチバイト文字に対応しているので、日本語も可能です。
半角から始まる数字、連続したハイフンから始まる名前はエラーになります。

Sample Code - 作成可能な変数

    $width10: 10px;
    $w-10: 10px;
    $w_10: 10px;
    $幅１０: 10px;
    $__w10: 10px;
    $-__-w: 10px;
    $良い感じの余白サイズ: 10px;

変数の補完

変数を使って一部値を参照する場合に、セレクタを参照や、画像等のファイルパスを指定する場合には、 そのまま記述すると、エラーになります。
ファイル名と変数名の区別がつかなくなるため、補完することで解決が可能です。

Sample Code - 作成可能な変数

    $selector: '.wrap,.innerWrap';
    $IMG_PATH: '../img/bg_main.jpg';
    #{$selector}{
        background: url(#{$IMG_PATH})
    }

ファイルパスやセレクタ名などは文字列として扱う為、""か''で囲う必要があります。
